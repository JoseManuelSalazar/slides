Grafos IV

Arboles de Expansión Mínima

Dado un grafo conexo, no dirigido y con pesos se debe seleccionar un subconjunto de arcos, de tal manera que el grafo continue siendo conexo y que el peso total de todos los arcos seleccionados sea mínimo.

Para poder cumplir con este criterio de conectividad, se deben elegir los V - 1 arcos que formen el arbol que cubra(conecte) todos los nodos V del grafo.

Un grafo puede tener más de un Arbol de Expansión Mínima con el mismo costo.

Algoritmos

  • Algoritmo de Kruskal
  • Algoritmo de Prim

Algoritmo de Kruskal

  1. Ordenar todos los arcos de menor a mayor peso.
  2. Tomar cada uno de los arcos ordenados y si no forma ciclo añadirlo. (La detección de ciclos puede ser facilmente implementada a traves de Disjoint Sets )
Ejemplo

Algoritmo de Prim

  1. Procesar: Tomar un nodo inicial, marcarlo como visitado.
  2. Procesar: Añadir a la cola de prioridad todos sus arcos (omitiendo aquellos arcos cuyo nodo destino ya haya sido visitado)
  3. Prim: Mientras que la cola de prioridad no se encuentre vacia, se evalua cada uno de los arcos que se encuentran en ella.
  4. Prim: Si el nodo del arco que se esta evaluando no ha sido visitado aún, se elige este arco y se procesa este nodo.

¿Cuándo usar Kruskal y cuándo Prim?

Hablemos de grafos densos y dispersos...

Grafos Densos

Les faltan muy pocas aristas de todas las posibles.

Grafos Dispersos

Tienen relativamente pocas aristas.

  • En grafos dispersos es conveniente usar el algoritmo de Kruskal
  • En grafos densos es conveniente usar el algoritmo de Prim
